3.1 ggplot

3.1.1 ggplot

ggplot2 is the tidyverse package meant to be used for all kinds of plot generation. Let’s look at how it works.

ggplot(titanic_train) +
  geom_point(mapping = aes(x = Age, y = Fare))

3.1.2 ggplot grammar

  • You begin a plot with the function ggplot(). It creates a coordinate system that you can add layers to.
  • The first argument of ggplot() is the dataset to use in the graph. So ggplot(titanic_train) creates an empty graph.
  • You complete your graph by adding one or more layers to ggplot()
  • To add layers, ggplot2 has provides a collection of “geoms.” Here, the function geom_point() adds a layer of points to your plot, which creates a scatterplot.
  • Each geom function in ggplot2takes a mappingargument. This defines how variabels in your dataset are mapped to visual properties.
  • The mappingargument is always paired with aes(), and the xand yarguments of aes()specigy which variables to map to the x and y- axes. ggplot2 looks for the mapped variable in the dataargument, in this cas, titanic_train.

3.1.3 ggplot grammar

The general grammar of ggplot will always be something like this, where you should replace the bracketed sections with a dataset, a geom function, or a collection of mappings:

ggplot(data = <DATA>) +
  <GEOM_FUNCTION>(mapping = aes(<MAPPINGS))

3.1.4 Aeshetic mappings

You can convey information about your data by mapping the aesthetics in your plot to the variables in your dataset. For example, you can map the colors of your points to the Pclass variable to reveal the class of each passenger:

ggplot(titanic_train) +
  geom_point(mapping = aes(x = Age, y = Fare, color = Pclass))

3.1.5 Facets

Another way to add additional variables, particularly useful for categorical variables, is to splot your plot into facets, subplots that each disply one subset of the data.

ggplot(titanic_train) +
  geom_point(mapping = aes(x = Age, y = Fare)) +
  facet_wrap(~ Pclass)

3.1.6 Position adjustments

ggplot(titanic_train) +
  geom_bar(aes(Pclass, fill = Survived))

3.1.7 Position adjustments

ggplot(titanic_train) +
  geom_bar(
    aes(Pclass, fill = Survived), 
    position = "fill")

3.1.8 Position adjustments

ggplot(titanic_train) +
  geom_bar(
    aes(Pclass, fill = Survived), 
    position = "dodge")

3.2 dplyr

3.2.1 Main dplyr functions

dplyr provides five key functions that allow you to solve the vast majority of your data-manipulation challenges :

  • pick observations by their values with filter()
  • reorder the rows with arrange()
  • pick variables by their names with select()
  • create new variables with mutate()
  • collapse many values down to a single summary with summarize()

3.2.2 dplyr grammar

All verbs work similarly :

  1. The first argument is a data frame
  2. The subsequent arguments describe what to do with the data frame, using the variable names (without quotes)
  3. The results is a new data frame

3.2.7 Managing NA

One important feature of R that can make comparison tricky is missing values, or NAs (“not availables”). NA represents an unknown value so missing values are “contagious”; almost any operation involving an unknown value will also be unknown.

NA > 5

x <- NA

is.na(x)

3.2.12 Select columns : helper functions

There are a number of helper functions you can use within select():

  • starts_with("abc") matches names that begin with “abc.”
  • ends_with("xyz") matches names that end with “xyz.”
  • contains("ijk") matches names that contain “ijk.”
  • matches("(.)\\1") selects variables that match a regular expression. Regular expression are a way to describe character patterns, we may have a chance to study them later.
  • num_range("x", 1:3) matches x1, x2, and x3.

See ?select for more details.

3.2.13 Add new variables

It’s often useful to add a new variables that are function of existing columns. That’s what the mutate() function does. In this example, we extract the family name from the Namevariable :

mutate(titanic_train, Fname = sub("\\,.*", "", Name)) %>% 
  select(PassengerId, Fname) %>% head()

3.2.14 Grouped summaries

The last key verb is summarize(). It collapses a data frame to a single row:

summarize(titanic_train, 
          fare = mean(Fare, na.rm = TRUE)
          )
##       fare
## 1 32.20421

3.2.15 Grouped summaries

summarize() is not terribly useful unless we pair it with group_by(). This changes the unit of analysis from the complete dataset to individual groups.

by_class <- group_by(titanic_train, Pclass)
summarize(by_class, 
          fare = mean(Fare, na.rm = TRUE)
          )

3.2.16 Combining operations with the pipe

As we saw just before, combining operations like group_by()and summarize() are useful, but not easy to write, because we has to create another object, by_class, to store the result of the first operation, before using it for the second operation.

3.2.17 Combining operations with the pipe

Another way to do this would be to use several functions at the same time, like this:

summarize(group_by(titanic_train, Pclass), 
          fare = mean(Fare, na.rm = TRUE)
          )

But this solution gets hard to write and read very fast when the number of operations you want to comine increases.

3.2.18 Combining operations with the pipe

That’s where the pipe comes in. It is an operator, written like this %>%

titanic_train %>% 
  group_by(Pclass) %>% 
  summarize(fare = mean(Fare, na.rm = TRUE))

3.2.19 Combining operations with the pipe

This focuses on the transformations, not what’s being transformed, which makes the code easier to read. You can read it as a series of imperative statements: group, then summarize. As suggested by this reading, a good way to pronounce %>% when reading code is “then.”

Behind the scenes, x %>% f(y) turns into f(x, y), and x %>% f(y) %>% g(z) turns into g(f(x, y), z), and so on. You can use the pipe to rewrite multiple operations in a way that you can read left-to-right, top-to-bottom. We’ll use piping frequently from now on because it considerably improves the readability of code. (Grolemung & Wickham, 2014, p. 65).
